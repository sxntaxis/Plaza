# 0.4.0 Commit Boundary - Demo Scaffold Baseline

## Purpose

`0.4.0` establishes the clean repository baseline for building the functional local Demo. It captures the Demo workspace, active contracts, state/signals scaffold, validation gate, MCP fail-closed guard, and scaffold tests without claiming that the Demo is complete.

This commit boundary exists so `0.5.0` can focus on executable Demo work instead of repeating research, layout decisions, and conceptual design.

## What 0.4.0 includes

- Root `demo/` workspace structure.
- Active Demo specs under `demo/specs/`.
- Demo ops docs under `demo/ops/`.
- Preserved research and audits under `demo/research/`.
- Source registry scaffold under `registry/`.
- JSON schema scaffold under `schemas/`.
- Ontology and shape scaffold under `ontology/`.
- Python package scaffold under `src/plaza/`.
- State/signals API scaffold: `emit_event`, `emit_issue`, `list_open_blockers`, `resolve_issue`.
- Artifact envelope scaffold with SHA-256 hash validation.
- Candidate-level publication gate scaffold.
- Validation report gate scaffold.
- MCP fail-closed startup guard.
- `doctor demo` CLI status command.
- Minimal `data/demo/canonical/demo.ttl` scaffold graph.
- Minimal `data/demo/validation/validation_report.json` scaffold report.
- Tests for scaffold behavior and guardrails.

## What 0.4.0 explicitly does not include

- Complete functional Demo.
- Real acquisition/reconciliation pipeline.
- Real four-resource legal dataset from preserved SCIJ artifacts.
- Full PROV-O chain from acquisition to canonicalization.
- Full SHACL engine validation.
- Full MCP JSON-RPC/FastMCP implementation.
- MCP tools/resources from the MCP profile.
- Citation-complete MCP responses.
- Claude Desktop smoke test.
- Public HTTP service, REST API, UI, SPARQL endpoint, Akoma Ntoso export, or public snapshots.

## Claims allowed after this commit

- Plaza has a committed Demo scaffold baseline.
- The Demo workspace has a stable root structure.
- The active Demo contracts are separated from foundational docs.
- Open blocker issues can prevent candidate publication in scaffold logic.
- MCP startup fails closed when validation is missing or nonconforming.
- The scaffold graph uses Demo ELI URIs and `eli:based_on` / `eli:basis_for` for the demonstration relation.

## Claims prohibited after this commit

- The Demo is functional for users.
- MCP tools/resources are implemented.
- The graph represents four real legal resources from preserved SCIJ artifacts.
- SHACL validation is fully integrated.
- Citation-complete responses are available.
- Plaza has completed a Claude Desktop local stdio smoke test.

## Files/directories expected in the commit

- `.gitignore`
- `README.md`
- `docs/`
- `demo/`
- `registry/`
- `schemas/`
- `ontology/`
- `data/demo/`
- `src/plaza/`
- `tests/`
- `pyproject.toml`

`archive/`, `var/`, `.git/`, Python caches, local DB files, and virtual environments are not part of this boundary.

## Pre-commit verification commands

```bash
git status --short
find docs -maxdepth 2 -type f | sort
find demo -maxdepth 4 -type f | sort
find src tests -type d -name "__pycache__" -o -type f -name "*.pyc"
grep -R "docs/demo" -n demo docs src tests data ontology schemas registry pyproject.toml README.md 2>/dev/null || true
grep -R "<old-demo-docs-path>\|<legacy-demo-eli-scheme>" -n demo docs src tests data ontology schemas registry pyproject.toml README.md 2>/dev/null || true
grep -R "plaza-demo://eli" -n demo docs src tests data ontology schemas registry pyproject.toml README.md 2>/dev/null || true
grep -R "https://demo.plaza.cr/eli" -n demo docs src tests data ontology schemas registry pyproject.toml 2>/dev/null || true
grep -R "eli:applies" -n demo docs src tests data ontology schemas registry pyproject.toml 2>/dev/null || true
grep -R "eli:based_on\|eli:basis_for" -n demo docs src tests data ontology schemas registry pyproject.toml 2>/dev/null || true
PYTHONPATH=src python -m plaza.demo.validate_graph
PYTHONPATH=src python -m plaza.cli.main doctor demo
PYTHONPATH=src python -m plaza.mcp.server
python tests/run_tests.py
PYTHONPATH=src python -m unittest discover -s tests
```

After tests, remove generated caches:

```bash
find src tests -type d -name "__pycache__" -prune -exec rm -rf {} +
find src tests -type f -name "*.pyc" -delete
```
